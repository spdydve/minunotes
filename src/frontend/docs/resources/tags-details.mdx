# Tags and note details

Use note details for built-in metadata and lightweight organization.

Open **Note actions → Details** from a note.

## Editable details

You can edit:

- Name
- Created date
- API editable
- Tags

Folder, type, updated date, and updated-by information are shown as read-only metadata.

## Tags

Tags are lightweight labels. They are useful for cross-cutting organization when folders are too broad.

Examples:

```txt
plan
research
oauth
release-notes
```

Tag names are normalized to lowercase words with optional dashes. For example, `Release Notes` becomes `release-notes`.

## Adding tags

Type a tag and press Enter, comma, or Add. Existing matching tags appear as suggestions while you type.

Tags are saved immediately when added or removed.

## Removing tags

Click the `x` on a tag pill to remove it from the note. If no notes use a tag anymore, it disappears from the reusable tag list.

## Not a database system

Tags are not custom properties. MinuNotes intentionally avoids a Notion-style database property system. Use note content, folders, wikilinks, backlinks, and tags for organization.
