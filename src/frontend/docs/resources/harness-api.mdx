# Harness API

The harness API is the core MinuNotes surface for agents, scripts, OpenAPI tools, and MCP adapters. API keys can access all non-private folders, selected project roots, or explicit specific folders.

## Auth

Send the API key with every request:

```http
X-API-Key: ntak_...
Content-Type: application/json
```

## Folder access

API keys can use one of two folder access modes:

- **All non-private folders**: access every non-private folder, including future non-private folders.
- **Project roots**: access selected top-level folders and their non-private subfolders.
- **Specific folders**: access only folders explicitly listed on the key.

Folder-level read-only blocks Create/Edit for All and Project-root scopes. Specific folder grants can be used as explicit write exceptions. Private folders are excluded from API, MCP, and integration access. A private parent also makes its child folders private for API access.

Folder creation is separate. Only keys with folder-creation permission can call:

```txt
POST /api/harness/folders
```

## Safe editing flow

1. Search or list folders.
2. Read the note.
3. Capture `contentHash`.
4. Send small edits with `baseHash`.
5. Read back the note and report final markdown or a changed-section summary.

## Edit payloads

```ts
type DocumentEdit =
  | { type: "append"; text: string }
  | { type: "replace_text"; oldText: string; newText: string }
  | { type: "replace_range"; from: number; to: number; text: string };
```

Prefer `replace_text` when the target text is unique. Use `replace_range` only after reading lines or sections.
