# Agent integrations

MinuNotes supports several integration paths for agents and tools. They all use the same scoped permissions model.

## Recommended paths

- **Harness API** for agents that can call HTTPS directly.
- **Hosted MCP** for MCP-native hosted agents and shared runtimes.
- **Local MCP** for desktop clients that spawn local MCP processes.
- **OpenAPI** for actions, REST tool importers, and compatibility with non-MCP platforms.

## Permissions model

Create one API key per agent or project. Keys can use **All non-private folders**, **Project roots**, or **Specific folders**.

Project roots are top-level folders that behave like lightweight workspaces for agents. A project-root grant includes that folder and its non-private subfolders. Specific folder grants are exact exceptions for a small set of folders.

Each key has Read, Create, and Edit permissions for its selected scope. Folder-level “Read-only for agents” blocks Create/Edit for All and Project-root keys, while Specific folder keys can be used as intentional write exceptions. Enable folder creation only when an agent is explicitly allowed to create folders.

Private folders are never accessible to agents or integrations. Private status also applies to child folders.

## Core endpoints

```txt
GET /api/harness/folders
POST /api/harness/notes
POST /api/harness/notes/:noteId/edit
GET /api/openapi.json
POST /api/mcp
```

Use `X-API-Key` on harness, OpenAPI-authenticated, and trusted/local hosted MCP requests. OAuth connected apps use `Authorization: Bearer mnoac_...` for hosted MCP and harness requests.
