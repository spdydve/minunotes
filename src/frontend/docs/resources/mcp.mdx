# MCP

MinuNotes supports both hosted MCP and local stdio MCP.

## Hosted MCP

Use hosted MCP for cloud agents, shared runtimes, and MCP-native integrations that can connect to a remote HTTP endpoint.

```txt
POST /mcp
GET /mcp
DELETE /mcp
```

Hosted MCP accepts either `X-API-Key: ntak_...` or `Authorization: Bearer mnoac_...` for OAuth connected apps, and routes tool calls through `/v1/harness/*`.

OAuth-capable hosted clients can discover authorization metadata from the MCP endpoint. MinuNotes advertises protected-resource metadata and Dynamic Client Registration (DCR), so clients such as ChatGPT can register a public OAuth client automatically instead of requiring manual app setup.

## Local MCP

Use local MCP for desktop clients that launch a process, such as local MCP-compatible tools.

```sh
export NOTES_API_URL="https://your-minunotes-host"
export NOTES_API_KEY="ntak_..."
notes-mcp
```

The monorepo can export a standalone MCP package artifact for a future `minunotes-mcp` mirror repo or npm package:

```sh
pnpm export:mcp
cd .release/minunotes-mcp
npm pack --dry-run
```

## Available tools

- `notes_list_folders`
- `notes_create_folder`
- `notes_search`
- `notes_get_note`
- `notes_create_note`
- `notes_edit_note`
- `notes_search_lines`
- `notes_read_lines`
- `notes_search_note_lines`
- `notes_read_section`

Tool discovery is handled by the MCP protocol. Clients call `tools/list`; MinuNotes registers tool names, descriptions, schemas, annotations, and handlers.
